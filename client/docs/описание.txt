Задача: разработать web конструктор для создания отчётов на базе старого open source проекта для Excel.

=============
Первый этап:
=============
Проект на figma - https://www.figma.com/file/Zj4rtg6ewHRdoAdQEzpXrd/Stat-Platform?node-id=0%3A1&t=31muaGOvqI0KAWpj-1
Он же в виде PDF - https://disk.yandex.ru/d/zMU3IFw85uBDtA

Характеристики отчётов: 
- редактируемые: название, принадлежность к отделу (не обязательно), личный/для всех. 
- статистические: автор и дата создания, последний редактор и дата модификации, популярность (кол-во открытий)

Личные отчёты - видны в списках/поиске только для текущего пользователя. Помечаются иконкой "личный".

Отделы: название, цвет (не обязательно). Цвет нужен для визуализации принадлежности отчёта к отделу (метка).

Пользователи: логин, пароль, роль.

Роли:
Админ: доступно создание/редактирование/удаление отчётов, отделов и пользователей (весь функционал). Видит чужие личные отчёты.
Редактор: доступно создание/редактирование/удаление отчётов. Не видит чужие личные отчёты.
Чтение: только просмотр общедоступных отчётов.

Главная страница: удобная навигация/поиск/сортировка отчётов, включая отчёты без отдела. Фильтр по отделам реализован как переход в папку с файлами.

Страница отдела: удобная навигация/поиск/сортировка отчётов конкретного отдела.

Профиль: удобная навигация/поиск/сортировка отчётов за авторством текущего пользователя.

Страница информации/редактирование отчёта: работа с редактируемыми характеристиками отчёта, просмотр статистических характеристик.

Планы на будущее развитие: 
- хранение и работа  с историей версий каждого отчёта, возможность восстанавливать из истории
- создание копий из доступных отчётов
- экспорт в Excel (только значения и оформление, без формул)
- больше ролей?
- страница автора (аналог профиля)?


=============
Второй этап:
=============
1. В качестве БД выступает PALO Server 3.0 (open source проект компании Jedox), общение через REST API. Планируется своя nodejs прослойка для этого. Пример API представлен на скриншотах.
 
2. Проект должен реализовать минимальный Exel-like функционал (таблица, работа со стоками/колонками, стилизация ячеек (цвет фона, стиль и цвет шрифта)) + PALO функции в ячейках для вставки данных. Например, 
=PALO.ENAME($A$1;"Measures";"Units";0;"Units")
=PALO.DATAC($A$1;$A$2;$A13;$A$3;$A$4;$A$5;$A$6;B$10;$A$7;$A$8)

3. Интерфейс пользователя должен повторять окна вставки отчёта, редактирования элементов измерений - см.скриншоты.
